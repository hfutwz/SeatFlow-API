# 预约规则与违约机制

## 预约规则

### 时间规则
- 以整点小时为单位（如 8:00-10:00）
- 单次最多4小时（系统可配置 max_reservation_hours）
- 必须在自习室开放时间内

### 冲突规则
- 同一时段同一座位不可重复预约
- 同一时段学生不可预约多个座位
- 院系权限：仅可预约有权限的自习室

### 预约生命周期
```
创建预约 → 待签到(PENDING) → 签到 → 使用中(CHECKED_IN) → 到时 → 已完成(COMPLETED)
                              ↓
                     超时15min未签到 → 已取消(CANCELLED) + 违约记录
                              ↓
                     主动取消 → 已取消(CANCELLED)
```

## 签到规则

### 签到时间窗口
- 开始时间：预约开始前15分钟
- 结束时间：预约结束时间
- 超出窗口不可签到

### 签到编码
- 每个教室每天生成一个动态编码
- 编码在教室现场屏幕上展示
- 每日更新

## 提醒与超时

### 三个时间节点
1. 预约开始前15分钟：推送提醒"预约即将开始"
2. 预约开始后10分钟未签到：推送警告"请尽快签到"
3. 预约开始后15分钟未签到：自动取消预约 + 记录违约 + 通知

### 取消规则
| 场景 | 结果 |
|---|---|
| 预约开始前取消 | 释放座位，不记违约 |
| 预约开始后、15分钟前取消 | 释放座位，不记违约 |
| 超时15分钟未签到 | 系统自动取消，记违约，不可手动取消 |

## 违约记录
- 类型：超时未签到(CHECK_IN_TIMEOUT)
- 由系统自动记录
- 学生和管理员均可查看
